# Workflow And Agents

## Main agent ownership

- Main agent отвечает за понимание задачи, план, интеграцию и финальный ответ.
- Не останавливайся на предложении, если пользователь явно просит реализацию и можно двигаться
  безопасно.
- Если встречаются чужие изменения, не откатывай их. Работай поверх них или обойди, если они не
  относятся к задаче.

## Subagents

Используй саб-агентов для независимых частей работы, которые можно делать параллельно без общего
write scope.

Типовые роли:

- Анализ запроса: уточнить требования, ограничения и недостающие данные.
- Планирование: разбить задачу на независимые шаги.
- Реализация: внести код в выделенной области ответственности.
- Review: проверить читаемость, SOLID/KISS/DRY, расширяемость и соответствие контракту.
- Verification: собрать проект, прогнать тесты, описать ошибки.

Правила:

- Саб-агент получает конкретную ограниченную задачу.
- Саб-агент не расширяет scope.
- Саб-агент не задает вопросы пользователю; при блокере возвращает вопрос main agent-у.
- Main agent ждет результаты нужных саб-агентов перед финальным ответом.

## TDD

Для новых фич и багфиксов:

- Сначала тест, который описывает желаемое поведение.
- Запусти тест и убедись, что он падает по ожидаемой причине.
- Затем добавь минимальную реализацию.
- Запусти тесты снова и проверь green.

Если тестовая инфраструктура уже есть, добавляй focused tests рядом с изменяемым модулем.

## Review and verification

- После существенных изменений выполняй code review-pass.
- Перед заявлением о готовности запусти свежую проверку, которая доказывает результат.
- В финальном ответе не говори, что тесты проходят, если свежий запуск не был выполнен.
