# Integrations And Secrets

## MCP integrations

В окружении могут быть доступны MCP-интеграции:

- Kaiten - карточки, контекст задач, комментарии.
- GitLab - merge requests, review comments, project metadata.
- Mobile - взаимодействие с Android/iOS окружением и устройствами.

Используй их только когда задача явно требует соответствующий внешний контекст или действие.

## Secrets

- Не записывай API tokens, personal access tokens, cookies, private keys или другие credentials в
  repo files.
- Если токен уже присутствует в пользовательском сообщении, не копируй его в документацию, код,
  тесты или логи.
- В документации описывай только назначение интеграции и способ безопасного обращения к ней.
- Если для задачи нужен секрет, ожидай его через безопасное окружение или уже настроенный
  MCP/tooling.

## Kaiten

Если пользователь просит контекст карточки или указывает Kaiten task/card id:

- используй доступный Kaiten MCP/tooling;
- верни id, title/name, description и релевантные comments;
- не сохраняй приватные данные карточки в repo без явного запроса.

## GitLab

Если пользователь просит исправить MR review comments:

- сначала найди MR текущей ветки через GitLab tooling;
- загрузи review comments с позициями;
- исправляй только релевантные замечания;
- после правок запусти релевантные проверки.
